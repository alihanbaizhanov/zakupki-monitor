# PRD: Система мониторинга статусов закупок

**Версия:** 1.0
**Дата:** 02.07.2026
**Автор/владелец продукта:** администратор бюджетных программ

---

## 1. Проблема и цель

Администратор бюджетных программ инициирует закупки товаров и услуг и сопровождает их до поставки совместно со службой закупа. Сейчас статус закупок не консолидирован в едином месте, из-за чего сложно:

- быстро увидеть, на каком этапе находится каждая закупка;
- выявить просроченные закупки (отставание от плановой даты);
- предоставить руководителю прозрачный обзор хода закупок без дополнительных отчётов.

**Цель продукта:** дать администратору единый реестр закупок с возможностью быстрого обновления статусов, а руководителю — режим наблюдения за ходом исполнения в реальном времени.

---

## 2. Пользователи и роли

| Роль | Кто | Права |
|---|---|---|
| **Администратор** | Администратор бюджетных программ | Полный доступ: добавление, редактирование и удаление закупок, изменение любых полей и статусов |
| **Руководитель (наблюдатель)** | Руководитель, курирующий закупки | Только просмотр: видит реестр, может пользоваться фильтрами, не может редактировать данные |

Переключение роли в MVP реализовано локально (без логина/пароля) как разграничение интерфейса. Полноценная аутентификация — см. раздел «Вне рамок MVP».

---

## 3. Пользовательские сценарии (user stories)

1. Как **администратор**, я хочу добавить новую закупку в реестр, чтобы зафиксировать начало процесса.
2. Как **администратор**, я хочу кликнуть по ячейке и быстро изменить статус, сумму, поставщика или дату, чтобы реестр всегда отражал актуальное состояние.
3. Как **администратор**, я хочу отфильтровать закупки по статусу/категории/способу закупки, чтобы найти нужные записи без прокрутки всей таблицы.
4. Как **руководитель**, я хочу открыть реестр в режиме просмотра и увидеть общую картину: сколько закупок в работе, сколько просрочено, сколько завершено.
5. Как **руководитель**, я хочу отфильтровать закупки, чтобы посмотреть, например, только «Работы» или только просроченные позиции.
6. Как **администратор**, я хочу видеть визуальное предупреждение по просроченным закупкам, чтобы вовремя реагировать.

## 4. Функциональные требования

### 4.1 Реестр закупок (таблица)
Каждая закупка — строка со следующими полями:

| Поле | Тип | Комментарий |
|---|---|---|
| № | авто-номер | не редактируется |
| Наименование закупки | текст | товар/услуга/работа |
| Категория | справочник | Товары / Работы / Услуги |
| Способ закупки | справочник | Конкурс / Аукцион / Запрос ценовых предложений / Из одного источника |
| Статус | справочник | см. п. 4.2 |
| Сумма, тенге | число | плановая сумма |
| Плановая дата | дата | дата, к которой планируется завершение этапа/поставка |
| Поставщик | текст | заполняется по мере определения победителя |
| № договора | текст | |
| Дата поставки (факт) | дата | фактическая дата поставки/исполнения |
| Ответственный | текст | инициатор/сопровождающий закупку |
| Комментарий | текст | произвольные заметки |

### 4.2 Справочник статусов
1. Планирование
2. Объявлен
3. Приём заявок
4. Рассмотрение заявок
5. Определение победителя
6. Заключение договора
7. Договор заключён
8. Поставка / исполнение
9. Завершено
10. Отменено

Каждый статус имеет цветовую индикацию для быстрого визуального распознавания.

### 4.3 Фильтры по столбцам
- Текстовый поиск: наименование, поставщик, № договора, ответственный, комментарий.
- Выпадающий список: категория, способ закупки, статус.
- Фильтр по дате: плановая дата, дата поставки.
- Кнопка «Сбросить фильтры».
- Фильтры комбинируются (логическое И).

### 4.4 Редактирование через всплывающее меню
- Доступно только администратору.
- Клик по редактируемой ячейке открывает всплывающее окно рядом с ячейкой с полем ввода, соответствующим типу данных (текст, число, дата, список).
- Действия: «Сохранить» / «Отмена», подтверждение по Enter, закрытие по Esc или клику вне окна.
- Руководитель не видит возможности клика по ячейкам как редактируемым.

### 4.5 Добавление и удаление закупок
- Кнопка «Добавить закупку» (только у администратора) создаёт новую строку с полями по умолчанию.
- Удаление строки — с подтверждающим диалогом (только у администратора).

### 4.6 Сводная панель (статистика)
Над таблицей отображаются агрегированные показатели:
- Всего закупок
- В работе (статус ≠ Завершено/Отменено)
- Просрочено (плановая дата прошла, статус не финальный)
- Завершено
- Суммарная плановая сумма

### 4.7 Индикация просроченных закупок
- Закупка считается просроченной, если плановая дата раньше текущей даты, а статус не «Завершено» и не «Отменено».
- Такие строки выделяются цветом и иконкой предупреждения.

### 4.8 Хранение данных
- Данные реестра — общие для всех пользователей, открывших приложение (единый источник правды для администратора и руководителя).
- Выбор роли сохраняется индивидуально для каждого пользователя/устройства.
- Данные сохраняются автоматически при каждом изменении.

## 5. Нефункциональные требования

- Интерфейс на русском языке.
- Работает в браузере без установки (веб-приложение).
- Адаптивность: таблица со сквозной прокруткой и зафиксированными заголовками для удобной работы с большим числом строк.
- Отклик интерфейса на действия пользователя — мгновенный (без перезагрузки страницы).

---

## 6. Вне рамок MVP (следующие итерации)

- Полноценная аутентификация пользователей (логин/пароль, различие «кто именно» из руководителей/администраторов).
- Ролевая модель с несколькими администраторами и историей изменений (аудит: кто и когда изменил статус).
- Уведомления/напоминания о приближающихся или просроченных плановых датах.
- Экспорт реестра в Excel/PDF.
- Прикрепление документов к закупке (техническое задание, договор, акты).
- Настраиваемый (администратором) справочник статусов и способов закупки без изменения кода.
- Графики и аналитика (динамика по месяцам, доля закупок по способу и т.д.).
- Мобильная версия/адаптивная вёрстка для смартфонов.

---

## 7. Метрики успеха

- 100% закупок администратора отражены в реестре в актуальном статусе.
- Снижение числа просроченных закупок за счёт своевременной визуальной индикации.
- Руководитель получает статус закупок без запроса отдельного отчёта у администратора.
